# 《Claude Code Anonymous》中文翻译

几周前，作者和老朋友 [Orta Therox](https://orta.io/) 重新联系上了。他们聊到 Claude Code 如何彻底改变自己构建软件的方式，以及自己已经很难把注意力放到别的事情上。

于是他们产生一个想法：做一个新的 meetup 形式——**Claude Code Anonymous**。

最开始有很多人想参加但不在伦敦。作者鼓励大家以“高能动性（High-Agency）”的方式，在自己所在城市复制这个活动。于是目前已经在这些城市有活动：
- [London](https://luma.com/u5rompg9)
- [Vienna](https://luma.com/q50cmcb2)
- [Berlin](https://luma.com/5lizqnpz)
- [Cologne](https://luma.com/j1fr97j3)
- [San Francisco](https://luma.com/i37ahi52)
- [Delft](https://luma.com/h5h322jz)
以及更多城市即将到来。

伦敦第一场非常成功，所以他们又安排了 10 月 13 日的下一场。

## 什么是 Claude Code Anonymous？
作者观察到：并不是所有人都会以同样方式“上头”。

有一类人（作者引用 Justin Searls 的概念）叫 **full-breadth developers**：既有技术能力也有产品能力的人，他们往往最快拥抱这种新型的软件构建方式。

作者半开玩笑说他们是“黑眼圈俱乐部”，因为他们经常睡不好。

“Claude Code Anonymous”这个名字就是在玩这种现象的梗。

他们想做一个空间，让大家能交流自己的体验，并且让人知道：对一些人来说，公开谈论自己使用 AI/LLM 工具仍然有压力。

因此：
- 演讲不录制
- 不直播

配方很简单：
- 找一群同频的人
- 提供场地
- 饮料、披萨
- 把重点放在社交与交流上

## Meetup 结构
这些 meetup 采用一系列 **闪电演讲（lightning talks）**：每个约 5 分钟。

演讲通常以这样的句式开头：
> “当 Claude Code 做了 Y 的时候，我感到 X。”

例如：
- “当 Claude Code 找到一个我追了很久的 bug 时，我很震撼……”
- “当 Claude Code 删掉了我的生产数据库时，我很失望……”

作者强调：
- Intro + Talks 一小时准时结束
- 留 2–3 小时给社交与交流

他参加过太多“讲太久导致大家讲完就走”的 meetup。
Claude Code Anonymous 的目的，是让你认识同频的人，而不是看长演讲。

5 分钟限制也能让更多人展示自己的成果，并作为未来做长演讲的起点。

## 为什么叫 Claude Code，而不是 X/别的名字？
不要太字面理解。

他们欢迎任何 agentic work，只是用 “Claude Code” 这个名字，是为了把受众对准“builder”，而不是市场/HR。

如果叫“agentic engineering”会吸引另一类人。

当然你也可以分享任何工具：codex、opencode、Cursor 等等。

作者个人观点：Claude Code 是最能代表这场革命的 agent，于是就把它当成了“专有名词的代称”（像 Jacuzzi、Tupperware 那样）。

如果你想保持中立，可以叫它 **Agents Anonymous**。

## 如何在你所在城市发起？
作者说很简单：
1) 找一个场地
2) 准备饮料
3) 宣传

他们用 [Luma](https://luma.com/) 做活动平台，并建议报名时问一些问题，比如：
- 你在做什么？
- 社交账号是什么？

目前每个活动的报名兴趣都远大于场地容量，所以他们会审核参与者，确保来的是“builder”，而不是来招人/营销产品的。

闪电演讲的规则也类似：
> 展示你学到了什么，而不是你要卖什么。

行为准则（Code of Conduct）他们用 CocoaPods 的那套，核心一句话就是：
- [“别当混蛋（Don’t be a Jerk）”](https://cocoapods.org/legal)

## 结尾
如果你想在你所在城市看到 Claude Code Anonymous：就去做！
