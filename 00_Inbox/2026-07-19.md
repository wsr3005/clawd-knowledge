# AI / Agent 战略情报｜2026-07-19

**一句话结论：** 今天真正值得改动作的不是又多了一个强模型，而是三件事同时发生：Kimi K3 成为可低成本实测的新执行层、Claude Code 获得一个月额度窗口、Agent 上线前的“上下文质量”开始有可复用的开源测量框架。

## 1. Kimi K3 已开放使用：值得进候选池，但必须按它的上下文约束单独建会话

【事实】月之暗面已正式推出 Kimi K3，并在 kimi.com、Kimi App、Kimi Work、Kimi Code 与 API 开放。官方规格为 2.8 万亿总参数、100 万 token 上下文；API 标价为每百万 token：缓存命中输入 2 元、未命中输入 20 元、输出 100 元。完整权重承诺在 2026-07-27 前发布。官方同时明确警告：Agent 框架若不完整回传历史思考、或从其他模型会话中途切到 K3，可能导致上下文干扰；模糊任务上可能过度自主，需要在 system prompt / AGENTS.md 明确边界。官方也承认整体体验仍落后于 Claude Fable 5 和 GPT-5.6 Sol。

【影响】这不是“立刻替换主力模型”的信号，而是一个很便宜、偏长程编码/视觉闭环的新执行层。对 Hermes、Codex、前端/3D/视频原型有测试价值；但若直接塞进现有混合模型长会话，恰好会踩中官方已知弱点，测试结果也会失真。

【动作】把 K3 作为隔离候选，不做默认替换：只在全新会话中跑固定任务集，保留完整历史，并用 AGENTS.md 收紧自主决策边界；优先测前端、3D、视觉反馈迭代和长程编码，架构/后端与百万字召回暂不依据社区帖子下结论。等 7 月 27 日权重与技术报告落地后，再评估本地部署成本和稳定性。

【证据】[Kimi 官方发布原文](https://mp.weixin.qq.com/s/lu1iJdvy_cDhVaX8PqbucQ)；[AI HOT 聚合入口（7 个独立信源，仅证明事件热度）](https://aihot.virxact.com/items/cmrodegdr021wbito32kpybwh)

【可信度/待核验点】产品可用范围、官方价格、上下文约束与权重发布日期为第一方已证实；性能、缓存率及案例效果均为厂商自测，尚未独立复现。社区流传的长上下文召回率、对比价格和榜单数字未纳入结论。

## 2. Claude Code 周额度提升延长至 8 月 19 日，Fable 5 的套餐规则同时落地

【事实】Claude 官方开发者账号宣布：Pro、Max、Team 与按席位 Enterprise 的 Claude Code 每周额度临时提高 50%，延长至 2026-08-19。Claude 官方另宣布，自 7 月 20 日起，Fable 5 纳入 Max 和 Team Premium，可用量为套餐限额的 50%；Pro 与 Team Standard 继续通过 usage credits 使用，并获一次性 100 美元 credit。

【影响】未来一个月是把高 token Claude Code 工作集中处理的明确窗口，但“周额度提高 50%”不等于 Fable 5 可以无限跑；Fable 的 50% 套餐额度规则仍会成为高强度任务的瓶颈。模型路由应按任务难度，而不是把所有开发统一迁回 Claude。

【动作】8 月 19 日前，把复杂重构、长程调试和高价值代码审查优先排入 Claude Code；普通实现继续由 Codex/低成本模型承担。Fable 5 只投到真正需要其判断力的任务，并记录每类任务的额度消耗，月底前决定套餐是否值得保留或升级。

【证据】[ClaudeDevs：Claude Code 周额度延长](https://x.com/ClaudeDevs/status/2078511173759324328)；[Claude：Fable 5 套餐与 credit 规则](https://x.com/claudeai/status/2078302415804379218)

【可信度/待核验点】两项均为官方账号直接公告，可信度高；实际账户显示、区域差异及 credit 到账情况仍应以控制台为准。

## 3. ProofAgent-Harness 把“上下文质量”做成 Agent 上线前检查项

【事实】新论文提出用七项标准测量 Agent 上下文：角色清晰度、护栏覆盖、指令一致性、工具 schema 质量、grounding 充分性、注入加固和 token 效率。作者称在固定模型、只改变上下文的受控实验中，这些指标可预测相应的下游行为；论文报告覆盖 7,500 个评估轮次。对应 ProofAgent-Harness 已开源，采用 Apache-2.0，支持多轮对抗评测、产物评分、pytest/CI 以及 Claude Code/Cursor 会话观察。

【影响】这直接补上 Hermes / Codex / Skills 工作流目前最容易被忽略的一层：别只测最终答案，还要在运行前审计 system prompt、工具定义、记忆、检索证据和不可信输入的组合质量。尤其适合把 prompt injection、工具误用和“确认后再变更”变成发布门禁，而不是靠事故后复盘。

【动作】先不把第三方评分当真理：抽取其七项标准，给现有关键 Agent/Skill 建一份轻量 preflight checklist；再选一个有工具调用和文件写入的工作流做本地 pilot，对比现有测试能否发现新增问题。只有 pilot 有增量价值，再接入 CI release gate。

【证据】[论文：AI Agents Do Not Fail Alone](https://arxiv.org/html/2607.14275v1)；[ProofAgent-Harness 开源仓库](https://github.com/ProofAgent-ai/proofagent-harness)；[Omar Sanseviero 推荐线索](https://x.com/omarsar0/status/2078585861315432645)

【可信度/待核验点】论文与仓库均可访问，开源能力已证实；“上下文指标可预测可靠性”及 7,500 轮结果来自作者自报，尚未独立复现，多评审 LLM 本身也可能引入偏差。
