## 方法论/经验

- Agent 记忆可以按 Trunk → Task → Skill → Recipe 抽象沉淀，并在召回时反向使用 Recipe/Skill 优先，减少碎片记忆冲突。 （@servasyy_ai）
  - https://x.com/servasyy_ai/status/2054406190667559259
- 区分 Context 与 Context Window：前者是 Agent 实际拥有并可工程化管理的信息，后者只是模型一次推理的容量限制。 （@dotey）
  - https://x.com/dotey/status/2054774181808488756
- 企业 Agent 落地的关键不是单次对话，而是把会议、客户沟通和决策理由及时写入长期记忆，避免员工事后转述造成上下文失真。 （@lxfater）
  - https://x.com/lxfater/status/2054462646527893681
- 做 AI 内容不能只看 AI 圈动态，还要从综艺、电影、小说和喜剧中学习节奏与反差，否则容易停留在转述和翻译。 （@Khazix0918）
  - https://x.com/Khazix0918/status/2054512737368318388
- 代码越来越廉价后，程序员需要重新评估技术、管理、营销和资本配置的权重，而不是只靠写代码本身建立护城河。 （@lxfater）
  - https://x.com/lxfater/status/2054831419848008184
