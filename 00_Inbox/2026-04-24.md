## 方法论/经验
- 一个很强的实用派观点：多数人不该自己折腾 LangChain/MCP 拼“垃圾 agent”，而该把任务材料集中进临时目录后，用 headless Claude Code + 脚本编排去跑完整任务链。 （@lidangzzz）
  - https://x.com/lidangzzz/status/2047247649456152790
- 另一个值得记的判断是：对大多数普通开发/调用工具任务，主流 LLM 的体感差距已远小于很多人的主观评价，别把少量样例结果误当成模型代际差。 （@lidangzzz）
  - https://x.com/lidangzzz/status/2047229429202469179
- 同一份 Skill 在 Claude Code、Codex 和其他 agent 上效果不同，本质是底层模型偏好与 harness/工具权限不同，因此更合理的写法是“能力感知 + 按环境分流”。 （@dotey）
  - https://x.com/dotey/status/2047388496944361834
