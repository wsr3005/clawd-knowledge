## 方法论 / 经验

- **Boris Cherny 的 Opus 4.7 实战建议很实用：少盯权限弹窗，多用 auto mode / recap / 自验证，把 agent 当长任务执行者而不是聊天对象。** 模型升级带来的生产力，往往来自工作流改造而不只是参数提升。  
  链接：https://x.com/dotey/status/2044868344256381254 （@dotey）

- **“Skill 的核心是分类和触发，而不是数量越多越好”这条判断很对。** 当技能数超过合理粒度后，命中率、速度和 token 成本都会恶化，设计 taxonomy 比堆技能更关键。  
  链接：https://x.com/Khazix0918/status/2044614662654984687 （@Khazix0918）

- **并不是所有任务都值得先写脚本再交给 Agent；真正的边界在于脚本复杂度是否已经高过直接让 Agent 执行。** 这提醒人机协作里最值钱的能力，仍然是先做正确的任务分解。  
  链接：https://x.com/lidangzzz/status/2044881648827744566 （@lidangzzz）

- **给 Agent 配独立邮箱这类“小基础设施”看起来琐碎，但很可能是把智能体真正嵌进工作流的关键一步。** 有身份、能收发、可留痕，Agent 才更像可持续协作对象。  
  链接：https://x.com/servasyy_ai/status/2044617303959228682 （@servasyy_ai）
