# 2026-07-18 AI / Agent 战略情报

**一句话结论：** 今天真正值得动手的不是追新品榜单，而是收紧 Codex 权限边界、把“可执行世界模型 + 回放验证”吸收到 Agent 架构，并将 Kimi K3 作为长上下文/视觉编码候选做隔离实测。

## 1. Codex Full Access 已出现误删 `$HOME`：把最小权限从建议升级为硬规则

【事实】OpenAI Core Products 负责人 Tibo Sottiaux 已确认：团队调查了少量 GPT-5.6 意外删除文件的报告；最常见条件是 Codex 开启 Full Access、没有沙盒保护且未启用 Auto Review，同时模型试图把 `$HOME` 改作临时目录却误删真实目录。OpenAI 正在更新 developer message、引导使用更安全权限模式，并增加 harness 级防护；详细复盘尚未发布。

【影响】这直接命中昱桥的 Agent/Codex 自动化边界。模型能力再强，也不能把“模型理解路径正确”当安全控制；一旦 Agent 拿到主目录级写权限，单次路径混淆就可能把可逆的软件错误变成不可逆的数据事故。

【动作】立即把默认规则定死：Codex/子 Agent 只获得任务目录权限，默认启用沙盒与 Auto Review；删除、批量移动、覆盖仓库外文件必须经过独立审批；临时目录使用系统生成的专用路径，不允许通过重写 `$HOME` 实现。OpenAI 复盘发布前，不把 Full Access 用于任何无人值守任务。

【证据】[OpenAI 负责人 Tibo Sottiaux 原始回应](https://x.com/thsottiaux/status/2077630111499882637)

【可信度/待核验点】高；事故模式与缓解措施来自 OpenAI 负责人。具体受影响人数、版本范围及最终 harness 修复仍待官方复盘。

## 2. Schema 把 ARC-AGI-3 公开集推到约 99%：Harness 设计的价值已不再是边角优化

【事实】Schema 项目报告：不改模型权重，而是强制 Agent 把状态与机制写成可编辑程序，用完整交互历史回放验证，再在模型内规划；Claude Opus 4.8/Fable 5 组合在 ARC-AGI-3 Public 上自报 98.98% RHAE，GPT-5.6 Sol 组合为 95.35%。项目明确标注这些 Public 结果尚未经 ARC Prize 验证；其 Claude 对照实验中，相同模型组合在通用 Claude Code scratch 基线为 42.83%。

【影响】核心信号不是“ARC 被刷榜”，而是 Agent 的外部认知结构可能比换模型更值钱：将隐式思考外化为可执行、可回放、可证伪的状态模型，能显著降低长任务中的幻觉累积。这个模式可直接迁移到 Hermes/OpenClaw/Kanban 自动化，而不是只用于游戏。

【动作】后续复杂自动化优先试三层结构：① append-only 事实时间线；② 可执行/可 diff 的任务状态与规则；③ 动作前全历史 backtest，现实反馈不符即废弃剩余计划。先在一个可重复验收的工作流上做 A/B，不把公开集自报成绩当通用能力证明。

【证据】[Schema 项目原文与方法、结果、限制](https://schema-harness.github.io/) · [AI HOT 聚合入口](https://aihot.virxact.com/items/cmropqooe05tqbitodvnqnc6u)

【可信度/待核验点】中高；方法、轨迹与数据公开，但 98.98%/95.35% 是项目方自报的公开集结果，尚无 ARC Prize 独立验证，也不能外推到开放世界任务。

## 3. Kimi K3 已可用：进入长上下文/视觉编码候选池，但不要替换 Fable 5 / GPT-5.6 Sol

【事实】月之暗面官方发布 Kimi K3：2.8T 参数 MoE、原生视觉、100 万 token 上下文；现已在 Kimi.com、Kimi Work、Kimi Code 与 API 提供，官方 API 价格为缓存命中输入 $0.30/MTok、未命中输入 $3/MTok、输出 $15/MTok，完整权重计划于 2026-07-27 发布。官方同时承认总体体验仍落后 Claude Fable 5 与 GPT-5.6 Sol，并警告：harness 若不回传完整 thinking history、或在会话中途切换模型，质量可能剧烈不稳定；模型也可能过度主动。

【影响】它不是“闭源旗舰平替”，但其长上下文、视觉闭环和编码定位值得成为仓库级理解、前端/3D、截图驱动修改的备选。真正的风险是只看 token 单价或榜单就换主力：输出价并不低，且 harness 兼容与过度主动会直接影响无人值守任务的成功成本和安全边界。

【动作】只做隔离轮换测试：新开会话，用一个真实仓库 + 截图 + 测试验收，记录一次成功成本、耗时、重试与人工修正；禁止中途从其他模型切到 K3，并在 `AGENTS.md` 写清目录、动作和停止边界。未通过真实任务 A/B 前，不替换 Fable 5 / GPT-5.6 Sol。

【证据】[Kimi K3 官方技术博客](https://www.kimi.com/blog/kimi-k3) · [AI HOT 多源聚合入口（18 个独立来源）](https://aihot.virxact.com/items/cmrnvwztt01bdbixyj6fk8322)

【可信度/待核验点】高（规格、可用性、价格、限制来自官方）；性能数据主要为厂商评测，完整权重与技术报告尚未发布，需等待 7 月 27 日并用自有任务复核。
